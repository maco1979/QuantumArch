分析：QuantumArch 参数量仅比同配置 Transformer 多 20-24%，而非预期的 2×。这是因为： (1) Cayley 参数化用 skew-Hermitian Ω（D² 个实参数）代替完整复数矩阵（2D² 个参数），节省了约一半参数； (2) QCI、QEL 等模块的额外参数占比很小。复数运算的实际开销在计算量（每次乘法 4 次浮点运算），而非参数量。

分析：QuantumArch 的梯度均值更大但中位数更小，说明梯度分布更不均匀（少数参数梯度很大）。 这可能是 Wirtinger 导数对模长和相位的分离优化导致的。QGD 优化器的模长/相位分离学习率设计正是为了应对这个问题。 最大梯度 4.38 需要注意——梯度裁剪（clip_grad_norm=1.0）在训练中是必要的。

关键发现：QuantumArch 在 CPU 上的延迟是 Transformer 的 200-4000 倍。主要原因是：
(1) PyTorch 复数支持不成熟：复数张量运算走 Python fallback 而非优化的 BLAS/cuBLAS 路径；
(2) 无 CUDA kernel：复数运算、Born 归一化、QFT 等没有专用 CUDA kernel；
(3) 多次张量转换：极坐标/直角坐标转换、Born 归一化等引入额外开销。
解决方案：Phase 2 计划中包含 CUDA kernel 优化 + 混合精度训练，预期可将延迟降至 Transformer 的 2-5×。

积极发现：QuantumArch 的延迟几乎不随序列长度增长（2.85-2.97ms），这验证了 QSA topk 模式的 O(n·k) 复杂度优势。 而 Transformer 的延迟随序列长度线性增长（1.78→3.68ms，符合 O(n²) 注意力）。 在 GPU 上且序列长度 > 512 时，QuantumArch 的延迟优势将更加明显。

分析：在随机初始化输入上，Born 概率分布接近均匀（H=3.76/4.16），远超所有阈值。 这是预期行为——QCI 早退机制在训练后的模型中才有效，因为训练会使某些 token 的表示变得确定（低熵）。 自适应阈值的设计意图是：训练初期保守（高 τ_low），后期逐步降低，让更多成熟 token 早退。 需要在真实训练后的模型上验证早退效果。

关键发现与下一步

  • 参数效率优秀：仅多 20-24% 参数即可实现 7 大量子启发机制，Cayley 参数化的参数效率很高。
  • 酉性约束完美：所有 Cayley 矩阵 ||W†W - I||_F = 0，严格满足酉性。
  • 序列扩展性优异：延迟几乎不随序列长度增长，验证了 O(n·k) QSA 的理论优势。
  • 初始表达能力：QA 初始损失低于 ST（4.60 vs 4.82），暗示复数表示的初始表达力更强。
  • CPU 延迟严重：复数运算在 CPU 上慢 200-4000×，需要 CUDA kernel + 混合精度优化。
  • 梯度分布不均匀：少数参数梯度很大（max 4.38 vs median 0.02），需要梯度裁剪。
  • 短训练 inconclusive：5 步训练不足以判断收敛性，需要 100+ epochs 的大规模实验。
  • QCI 早退待验证：需要在训练后的模型上评估早退机制的实际效果。
